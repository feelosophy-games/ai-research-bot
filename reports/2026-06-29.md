### [OpenAI, GPT-5.6 Sol·Terra·Luna 한정 프리뷰 공개](https://openai.com/index/previewing-gpt-5-6-sol/)
- **요약**: 6월 26일 OpenAI가 차세대 모델군 GPT-5.6을 한정 프리뷰로 공개했다. 최상위 Sol(코딩·생물학·사이버보안 에이전트 성능 강화), 일상 업무용 Terra(GPT-5.5급 성능에 가격 절반), 저비용·고속 Luna 3티어 구성이다. 가격은 1M 토큰당 Sol $5/$30, Terra $2.50/$15, Luna $1/$6. 미 정부와 사전 공유 후 약 20개 조직에만 우선 제공하며 수 주 내 일반 출시 예정.
- **왜 중요**: 숫자(세대)와 이름(Sol/Terra/Luna = 지능/속도/비용 티어)을 분리한 새 네이밍은 모델 선택을 단순화한다. 같은 성능을 더 싸게 주는 흐름(Terra)이 가격 경쟁의 방향을 보여준다.
- **업무 적용 포인트**: 작업을 난이도별로 분류해 Sol/Terra/Luna에 라우팅하면 품질을 지키면서 비용을 크게 줄일 수 있다. 일반 출시 전까지는 GPT-5.5 기반 파이프라인을 유지하며 마이그레이션 계획만 미리 세워둔다.

### [ByteDance, Doubao Seed 2.1 Pro·Turbo 출시](https://dataconomy.com/2026/06/24/bytedance-launches-doubao-2-1-pro-language-model/)
- **요약**: 6월 24일 ByteDance가 에이전트 특화 모델 Seed 2.1을 Pro/Turbo 두 버전으로 공개했다. Pro는 장기 계획·자가 수정에 강한 플래그십 추론 모델, Turbo는 대규모 프로덕션용 저지연·저비용 모델이다. 코딩 엔지니어링, 장기 체인 에이전트 실행, 멀티모달 이해 세 방향을 집중 개선했고 GPT-5.5급 코딩·에이전트 성능을 주장한다. Pro 가격은 입력 100만 토큰당 6위안, 출력 30위안(캐시 적중 시 1.2위안).
- **왜 중요**: 중국 프런티어 모델들이 DeepSeek V4의 가성비 벤치마크에 맞춰 빠르게 추격 중이며, "단발 답변"이 아니라 "다단계 작업 완수"를 목표로 한 에이전트 지향 설계가 표준이 되고 있다.
- **업무 적용 포인트**: 비용 민감한 대량 에이전트 워크로드에서 Turbo가 후보가 될 수 있다. 다만 데이터 거버넌스·리전 제약을 먼저 확인하고 PoC로 코딩·장기작업 정확도를 검증한 뒤 도입한다.

### [Sakana AI, 멀티에이전트 오케스트레이션 모델 Fugu Ultra 출시](https://sakana.ai/fugu-release/)
- **요약**: 6월 22일 일본 Sakana AI가 여러 프런티어 모델을 단일 API 뒤에 묶는 멀티에이전트 시스템 Fugu의 최상위 버전 Fugu Ultra를 출시했다. ICLR 2026 논문(Trinity, Conductor) 기반의 학습된 모델 오케스트레이션으로, SWE-Bench Pro에서 73.7%를 기록해 Claude Opus 4.8(69.2%)·GPT-5.5(58.6%)·Gemini 3.1 Pro(54.2%)를 앞섰다. 구독 $20/월(Fugu·Ultra 포함), Ultra API는 1M 토큰당 $5/$30.
- **왜 중요**: "모델 하나"가 아니라 "에이전트 풀을 상황에 맞게 라우팅"하는 접근이 단일 모델 한계를 넘는 방향을 제시한다. 교체 가능한 에이전트 풀은 특정 벤더 API 접근이 끊겨도 우회할 수 있어 공급 리스크 대응책이 된다.
- **업무 적용 포인트**: 난이도 높은 다단계 문제(복잡한 코드 수정 등)에 오케스트레이션형 서비스를 시도해볼 만하다. 단일 벤더 종속을 줄이려는 조직은 라우팅 계층 도입을 검토한다.

### [Salesforce, Agentforce Commerce 에이전트 3종 정식 출시](https://www.digitalcommerce360.com/2026/06/24/salesforce-releases-ai-agents-b2b-ecommerce-updates/)
- **요약**: 6월 24일 Salesforce가 역대 최대 규모의 Agentforce Commerce 업데이트로 Shopper·Buyer·Merchant 에이전트를 정식(GA) 공개했다. Shopper는 스토어프런트에서 탐색→결제→CS까지 대화로 처리(초기 고객사 전환율·장바구니 담기 두 자릿수 개선 보고), Buyer는 WhatsApp·SMS에서 B2B 발주("3월 주문과 동일하게 16oz 패스너 40케이스")를 처리, Merchant는 자연어로 카탈로그 운영을 맡는다. 여름 중 ChatGPT·Google Search(AI Mode)·Gemini 앱 연동 예정.
- **왜 중요**: 에이전트가 데모 단계를 넘어 실제 매출 채널(커머스)에 GA로 투입된 대표 사례다. AI 추천 트래픽이 소셜 대비 8배 전환율을 보인다는 데이터는 채널 전략 자체를 바꾼다.
- **업무 적용 포인트**: 커머스 운영팀은 카탈로그 관리·상품 정렬 같은 반복 업무를 자연어 에이전트로 위임해 시간을 절감할 수 있다. 도입 시 "탐색→결제" 같은 경계가 명확하고 측정 가능한 워크플로우부터 시작한다.
