# AI 리서치 봇 — 설계 문서

작성일: 2026-06-29

## 1. 개요

주 1회(금요일 17:00 KST) 자동으로 다음 두 주제를 웹 리서치해, "요약 + 인사이트" 형식의
마크다운 리포트를 전용 git 저장소에 날짜별로 누적하는 봇.

- **주제 1**: AI 활용 / 업무 자동화 사례 (실무 적용 레퍼런스·워크플로우)
- **주제 2**: AI 모델 / 제품 뉴스 (신규 모델 출시, 주요 업데이트, 업계 동향)

목적: 업무 효율에 도움이 되는 최신 정보를 주기적으로 손쉽게 받아보기.

## 2. 핵심 결정 사항

| 항목 | 결정 |
|------|------|
| 출력 형태 | 마크다운 파일 (`reports/YYYY-MM-DD.md`) |
| 실행 주기 | 주 1회, 금요일 17:00 KST |
| 실행 방식 | 클라우드 루틴 (cron) — PC 꺼져 있어도 실행 |
| 저장소 | 새 전용 repo `jp-projects/ai-research-bot` + GitHub 원격 |
| 리포트 깊이 | 요약 + 인사이트 (왜 중요한지 / 업무 적용 포인트) |

## 3. 구성 요소 (역할 분리)

- **리서치 프롬프트** (`research-prompt.md`)
  봇이 매번 실행하는 지시문. 주제·검색 범위(지난 1주)·출력 형식을 고정.
  스케줄러와 독립적으로 단독 실행/테스트 가능 — 이것이 봇 로직의 단일 진실 공급원.

- **리포트 템플릿**
  각 주차 출력 마크다운의 고정 구조 (섹션 5 참조). 프롬프트가 이 형식을 강제.

- **스케줄러**
  `schedule` 스킬로 등록하는 주간 클라우드 루틴(cron, 금 17:00 KST).
  매주 리서치 프롬프트 실행 → 리포트 생성 → 저장소에 커밋/푸시.
  > 주의: 클라우드 루틴이 저장소에 접근·푸시하는 정확한 메커니즘은 빌드 시
  > `schedule` 스킬로 확정한다. (추측 박제 금지)

- **저장소** `jp-projects/ai-research-bot`
  - `research-prompt.md` — 리서치 지시문
  - `reports/YYYY-MM-DD.md` — 주차별 결과 누적
  - `docs/superpowers/specs/` — 본 설계 및 후속 계획 문서

## 4. 데이터 흐름

```
(금 17:00 KST) cron 트리거
  → 리서치 프롬프트 실행 (WebSearch/WebFetch로 지난 1주 자료 수집)
  → 출처 정리 + 중복/저품질 필터 + 요약·인사이트 작성
  → reports/YYYY-MM-DD.md 생성
  → git add / commit / push
  → (사용자 맥에서) git pull 로 확인
```

## 5. 리포트 형식 (요약 + 인사이트)

```markdown
# AI 리서치 리포트 — YYYY-MM-DD (이번 주)

## 🗞️ AI 모델 / 제품 뉴스
### [제목](링크)
- **요약**: 무슨 일인지 2~3줄
- **왜 중요**: 인사이트
- **업무 적용 포인트**: 어떻게 써먹을지

## 🛠️ AI 활용 / 업무 자동화 사례
### [제목](링크)
- **요약**: ...
- **왜 중요**: ...
- **업무 적용 포인트**: ...

## 한 줄 정리
이번 주 핵심 3가지
```

## 6. 에러 처리 / 견고성

- **검색 결과 빈약**: 항목 수가 적어도 수집된 만큼만 작성하고 부족함을 명시 (빈 리포트 방지).
- **푸시 실패**: 루틴 로그에 기록. 누락분 자동 복구는 하지 않음(단순 유지). 대신
  리포트 상단에 대상 기간을 명시해 사람이 추적 가능하게 함.
- **중복·저품질 출처**: 프롬프트 지시로 필터링.

## 7. 범위 밖 (YAGNI)

- 이메일 / Slack / Notion 알림 — 하지 않음 (마크다운 파일만)
- 항목 자동 점수화·랭킹 — 하지 않음
- 과거 리포트 자동 요약/롤업 — 하지 않음

## 8. 성공 기준

- 매주 금 17:00 KST에 자동으로 새 리포트가 저장소에 생성·푸시된다.
- 각 리포트가 두 주제를 "요약 + 왜 중요 + 업무 적용 포인트" 형식으로 담는다.
- 리서치 프롬프트를 수동 실행해도 동일한 형식의 리포트가 나온다.
